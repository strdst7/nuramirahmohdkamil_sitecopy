Part 2: Software Design Document (SDD)Architecture Focus: Serverless, AI-First, Scalable Lead Generation.1. Technology StackFrontend & Framework: Next.js (App Router), React, Tailwind CSS (for rapid styling of the grid/blueprint aesthetic).Hosting: Vercel (seamless Next.js deployment and Edge functions for fast AI responses).Backend / API: Next.js API Routes (Serverless functions to securely call external AI APIs without exposing your keys).Database: Supabase (Serverless PostgreSQL). Handles lead capture and waitlists.2. AI Integration ArchitectureYour frontend will never call OpenAI or Gemini directly. It will call your Next.js backend, which will securely handle the prompts.OpenAI Compatible Endpoint: POST /api/ai/openaiHandles requests for text generation, structuring data, or logic processing.Gemini Endpoint: POST /api/ai/geminiLeveraged for multimodal tasks (if you want users to upload images/documents to test) or fast text reasoning.Fallback/Routing Logic: You can build a utility that routes the request to whichever API is faster or cheaper for that specific playground tool.3. Database Schema (Supabase / Postgres)We will keep it incredibly lightweight. We just need to track who is interested in what.Table: playground_leadsColumnTypeDescriptionidUUIDPrimary Keyproject_nameStringe.g., "Experiment 04: Document Analyzer"emailString (Nullable)User's emailwhatsappString (Nullable)User's WhatsApp numberintentEnum'white-label', 'update-notification', 'beta-access'created_atTimestampAuto-generated4. Component Architecture (High-Level)Layout.tsx: Contains the global grid background, top navigation bar, and footer with LinkedIn/Email links.PlaygroundDemo.tsx: The interactive wrapper for your 7 tools. Handles API state (Loading, Success, Error).LeadCaptureForm.tsx: A reusable, minimalist form component that triggers a POST request to Supabase to save user interest.
