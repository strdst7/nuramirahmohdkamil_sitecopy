**Step 1: Create `src/app/api/newsletter/unsubscribe/route.ts`**

Create the POST handler following the exact pattern from `src/app/api/leads/route.ts` and the subscribe route from Plan 02. Per D-16: server-side, validates UUID, updates status, returns JSON.

**Zod schema for request body:**
```typescript
import { z } from "zod";
const unsubscribeSchema = z.object({
  token: z.string().uuid("Invalid unsubscribe token"),
});
```

**Handler logic:**

1. Parse body with `unsubscribeSchema.safeParse(body)`
2. On validation failure: `return Response.json({ error: "Invalid token" }, { status: 400 })`
3. Query `newsletter_subscribers` WHERE `id` = parsed token:
   ```typescript
   const { data: subscriber, error: lookupError } = await getSupabase()
     .from("newsletter_subscribers")
     .select("id, email, status")
     .eq("id", token)
     .single();
   ```
4. If `lookupError` or `subscriber === null` (token not found): 
   `return Response.json({ error: "not_found", message: "This link doesn't look right — check your email for the correct link." }, { status: 404 })`
5. If `subscriber.status === 'unsubscribed'`:
   `return Response.json({ status: "already_unsubscribed", message: "You're already unsubscribed from this newsletter." }, { status: 200 })`
6. If `subscriber.status === 'active'`: Update status to 'unsubscribed' per D-08 (soft delete):
   ```typescript
   const { error: updateError } = await getSupabase()
     .from("newsletter_subscribers")
     .update({ status: "unsubscribed", updated_at: new Date().toISOString() })
     .eq("id", token);
   ```
   On success: `return Response.json({ status: "unsubscribed", message: "You've left the echo. You can re-subscribe anytime." }, { status: 200 })`
   On update failure: `return Response.json({ error: "Failed to update subscription" }, { status: 500 })`

**Do NOT send any email on unsubscribe (per D-05).**

**Full response contract:**
```json
// Successful unsubscribe
{ "status": "unsubscribed", "message": "You've left the echo. You can re-subscribe anytime." }

// Already unsubscribed (200 - per D-15, friendly)
{ "status": "already_unsubscribed", "message": "You're already unsubscribed from this newsletter." }

// Token not found (404 - per D-15, friendly)
{ "error": "not_found", "message": "This link doesn't look right — check your email for the correct link." }

// Invalid UUID format (400)
{ "error": "Invalid token", "fieldErrors": { "token": ["Invalid unsubscribe token"] } }

// Server error (500)
{ "error": "Failed to update subscription" }
```

**Important:** Use `Response.json()` for all responses. The 200 status on "already_unsubscribed" is intentional — it's not an error condition, just an idempotent action.

**`handleUnsubscribe` logic:**
```typescript
const handleUnsubscribe = async () => {
  setIsUnsubscribing(true);
  try {
    const res = await fetch("/api/newsletter/unsubscribe", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ token }),
    });
    const data = await res.json();
    if (data.status === "unsubscribed") {
      setPageState("done");
    } else if (data.status === "already_unsubscribed") {
      setPageState("already_unsubscribed");
    } else {
      setPageState("error");
      setErrorMessage(data.message || "Something went wrong");
    }
  } catch {
    setPageState("error");
    setErrorMessage("The echo is quiet right now. Please try again.");
  } finally {
    setIsUnsubscribing(false);
  }
};
```

**Effect hook for initial token validation (optional optimization):**
On mount, if token is null/empty, immediately show error state rather than loading. If token is present but doesn't look like a UUID (simple regex check: `^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$`), immediately show error. This avoids an API call for obviously invalid tokens.

```typescript
useEffect(() => {
  if (!token || token.length === 0) {
    setPageState("error");
    setErrorMessage("No unsubscribe token found. Check your email link.");
    return;
  }
  // Validate UUID format client-side before calling API
  const uuidRegex = /^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i;
  if (!uuidRegex.test(token)) {
    setPageState("error");
    setErrorMessage("This link doesn't look right — check your email for the correct link.");
    return;
  }
  setPageState("confirm");
}, [token]);
```
