**Step 1: Create `supabase/migrations/001_newsletter_subscribers.sql`**

Create the migration directory and file. The SQL defines the `newsletter_subscribers` table exactly per D-07:

```sql
-- Migration: Create newsletter_subscribers table
-- Phase 6: Engagement — Newsletter subscription tracking
-- Run this in Supabase SQL Editor or via supabase CLI

CREATE TABLE IF NOT EXISTS newsletter_subscribers (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  email text NOT NULL UNIQUE,
  status text NOT NULL DEFAULT 'active' CHECK (status IN ('active', 'unsubscribed')),
  subscribed_at timestamptz NOT NULL DEFAULT now(),
  updated_at timestamptz NOT NULL DEFAULT now()
);

-- Index for fast email lookups (subscribe/duplicate check and unsubscribe)
CREATE INDEX IF NOT EXISTS idx_newsletter_subscribers_email ON newsletter_subscribers (email);

-- Index for listing active subscribers
CREATE INDEX IF NOT EXISTS idx_newsletter_subscribers_status ON newsletter_subscribers (status);
```

**Design decisions per CONTEXT.md:**
- D-06: Separate table from `playground_leads` — different domain
- D-07: id (UUID PK with gen_random_uuid()), email (text UNIQUE), status (text DEFAULT 'active' with CHECK constraint), subscribed_at (timestamptz DEFAULT now()), updated_at (timestamptz DEFAULT now())
- D-08: Soft delete via `status = 'unsubscribed'` — rows are never physically deleted
- D-09: Unsubscribe token is the row's UUID `id` — no separate token column needed
- No RLS needed per CONTEXT.md OpenCode's Discretion (server-side access only via service role)

**Step 2: Create `src/app/api/newsletter/subscribe/route.ts`**

Create the POST handler following the EXACT pattern from `src/app/api/leads/route.ts`:

- Zod validation for `{ email: string }`
- Supabase upsert-or-check pattern for duplicate detection (per D-08 — if already unsubscribed, re-subscribe is blocked)
- On success: insert, send confirmation email via `sendSubscribeConfirmation()`, return 200
- On duplicate (unique constraint violation): return 200 with `{ status: "already_subscribed", message: "You're already in the echo" }` (per D-12 — friendly, not an error)
- On already-unsubscribed (status='unsubscribed'): return 200 with `{ status: "previously_unsubscribed", message: "This email was previously unsubscribed" }` (soft block per D-08 — prevents re-subscription after opt-out)

**Zod schema:**
```typescript
const subscribeSchema = z.object({
  email: z.string().email("Please provide a valid email address").min(1, "Email is required"),
});
```

**Duplicate handling approach (per D-08):**
Instead of just catching the UNIQUE constraint error, query first:

1. SELECT from `newsletter_subscribers` WHERE email = parsed.data.email
2. If row exists AND row.status === 'active': return `{ status: "already_subscribed", message: "You're already in the echo" }` with 200 (per D-12)
3. If row exists AND row.status === 'unsubscribed': return `{ status: "previously_unsubscribed", message: "This email was previously unsubscribed. Contact us to re-subscribe." }` with 200
4. If no row exists: INSERT with email, status='active'. On success, call `sendSubscribeConfirmation(email)` fire-and-forget (with `.catch()`). Return `{ status: "subscribed", message: "Welcome to the echo." }` with 200.

**Fire-and-forget email call (per D-05 — confirmation email only):**
```typescript
import { sendSubscribeConfirmation } from "@/lib/email";
// After successful insert:
sendSubscribeConfirmation(email).catch((err) => console.error("Subscribe confirmation failed:", err));
```

**API call logic:**
```typescript
const handleSubmit = async (e: FormEvent) => {
  e.preventDefault();
  if (!email.trim()) return;
  setIsSubmitting(true);
  setError(null);
  try {
    const res = await fetch("/api/newsletter/subscribe", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ email: email.trim() }),
    });
    const data = await res.json();
    if (data.status === "subscribed") {
      setStatus("success");
    } else if (data.status === "already_subscribed") {
      setStatus("already_subscribed");
    } else if (data.status === "previously_unsubscribed") {
      setStatus("previously_unsubscribed");
    } else {
      setError(data.message || "Something went wrong");
    }
  } catch {
    setError("The echo is quiet right now. Try again in a moment.");
  } finally {
    setIsSubmitting(false);
  }
};
```
